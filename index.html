◇

Agents Window

全新 Agent 优先界面，支持本地、worktree、云端、SSH 多环境并行运行。侧栏统一管理所有 Agent，包括从移动端、Web、Slack、GitHub、Linear、Jira、Microsoft Teams 发起的任务。云 Agent 自动生成演示和截图供验证。

  • 多仓库原生支持
  • 本地↔云端无缝切换
  • 可随时切回经典 IDE
◎

Design Mode

在内置浏览器中直接标注、框选 UI 元素，精准反馈给 Agent。支持快捷键操作：⌘⇧D 切换、⇧拖拽选区、⌘L 加入对话。

▦

Agent Tabs

多会话并排或网格查看，方便对照不同方案。支持 /multitask 异步子代理并行处理，将大任务拆分同时执行。

⎔

Auto-review 智能审查

3.6 新增：全新运行模式，减少审批疲劳。白名单命令立即执行，沙盒任务安全运行，其他请求由分类子 Agent 自主决策是否放行。

↻

/loop 循环任务

3.5 新增：按计划重复运行提示，直到达成目标或手动停止。适合监控部署状态、持续测试等长时间任务。

⎘

MCP 结构化输出 & 插件市场

工具返回更丰富结构数据。Marketplace 提供数百款插件（MCP、Skills、Subagents、Rules、Hooks），一键安装。团队可创建私有市场，配置分发策略：默认关闭 / 默认开启 / 强制安装。

版本演进

大版本升级脉络

下面是官方叙事里的几个「分水岭」：从编辑器里加 AI，到多 Agent 并行，再到全新 Agent 优先界面，现在进入智能审查与自动化时代。

  1. 1.x 时代 · 底座

    AI 原生编辑器成型

    Cursor 选择 fork VS Code 而非只做插件，以便深度改造交互与模型管线。根据 2.0 changelog 的追溯：面向 Agent 的内置浏览器与沙盒终端在 1.7 即以 beta 形态出现。

    • 代码补全、对话、代码库索引等日常能力持续迭代
    • 企业场景逐步补齐：终端沙箱、审计、团队规则等
  2. 2.0 · 2025 年 10 月 29 日

    Composer 与多 Agent 工作台

    Changelog 2.0 将产品明确推到「Agent 并行」：侧栏管理 Agent 与 Plan，同一提示下最多 8 个 Agent 并行。

    • Composer：Cursor 自研的 agentic 编码模型，强调低延迟与多轮迭代
    • Browser GA：内置浏览器正式版，可选中 DOM 元素并把上下文回传给 Agent
    • Sandboxed Terminals GA（macOS）：默认在沙箱执行未白名单命令
    • Plan Mode：规划与执行可拆分模型
    • Voice Mode：语音驱动 Agent；Cloud Agents 体验增强

    阅读 2.0 发布博文

  3. Composer 2 · 2026 年 3 月 19 日

    新一代自研模型（承上启下）

    官方在 Composer 2 中将其定位为新的前沿编码模型；与 3.0 博文中的「云 ↔ 本机交接时快速迭代」叙事衔接。

  4. 3.0 · 2026 年 4 月 2 日

    界面重构：Agent 优先的统一工作区

    Cursor 3 强调从底层重做界面：天然 multi-workspace / 多仓库，桌面与 cursor.com/agents 的云 Agent 体验合一。

    • 与 2.0 的关系：2.0 把「多 Agent + 模型 + 浏览器工具」做深；3.0 把「编排面」升级为独立 Agents Window
    • 可随时回到经典 IDE；也可本地/云端会话互迁

编辑器

Agent Tabs 与命令

Agent Tabs 支持并排或网格查看多会话。/worktree 与 /best-of-n 成为编辑器内新入口；/loop 支持循环任务；/multitask 支持异步并行。

  • /worktree：单独 git worktree，实验性改动与主分支隔离。
  • /best-of-n：多模型并行尝试同一任务，各自 worktree，便于横向对比质量。
  • /loop：按计划重复运行，直到达成目标。适合监控和长时间任务。
  • /multitask：异步子代理并行处理，将大任务拆分同时执行。
  • 大文件 diff 渲染更快、更省内存；Agent 对长耗时任务监控更稳。
  • 新增 Await 工具：可等待后台 shell、子 Agent 或特定输出，减少盲等重试。
