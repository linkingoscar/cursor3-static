Cursor 3.x

Highlight Reel

◇

Agents Window

New Agent-first interface supporting parallel execution across local, worktree, cloud, and SSH. Unified sidebar manages all Agents, including those from mobile, web, Slack, GitHub, and Linear. Cloud Agents auto-generate demos and screenshots for verification.

  • Native multi-repo support
  • Seamless local↔cloud handoff
  • Switch back to classic IDE anytime
◎

Design Mode

Annotate and select UI elements directly in the built-in browser for precise Agent feedback. Keyboard shortcuts: ⌘⇧D to toggle, ⇧+drag to select, ⌘L to add to chat.

▦

Agent Tabs

Side-by-side or grid view for comparing different approaches. Supports /multitask async subagents to parallelize and break down larger tasks simultaneously.

⎇

/worktree

Create isolated git worktrees for experimental changes separate from main branch. Run multiple background tasks, then bring any branch to foreground with one click.

≋

/best-of-n

Multiple models attempt the same task in parallel, each in its own worktree. Compare results and merge the best solution. Ideal for exploratory tasks and quality comparison.

⎘

MCP Structured Output & Marketplace

Tools return richer structured data. Marketplace offers hundreds of plugins (MCP, Skills, Subagents, Rules, Hooks) with one-click install. Teams can create private marketplaces with distribution policies: Default Off / Default On / Required.

Major Version Progression

  1. 1.x Era · Foundation

    AI-native Editor Takes Shape

    Cursor chose to fork VS Code rather than just making a plugin, enabling deep transformation of interaction and model pipelines.

  2. 2.0 · October 29, 2025

    Composer & Multi-Agent Workbench

    Explicitly pushed the product to "Agent parallelism": up to 8 Agents in parallel, with Composer, Browser GA, Sandboxed Terminals, Plan Mode, and Voice Mode.

    Read 2.0 Release Blog

  3. Composer 2 · March 19, 2026

    New-gen In-house Model (Bridge)

    Officially positioned as the new frontier coding model; connects with 3.0's "fast iteration during cloud↔local handoff" narrative.

  4. 3.0 · April 2, 2026

    Interface Rebuild: Agent-first Unified Workspace

    Emphasizes rebuilding the interface from the ground up: natively multi-workspace / multi-repo, desktop and cloud Agent experience unified; Design Mode brings browser annotations into Agent.

Favoring Cursor

Long-term in VS Code-like interfaces; need LSP, built-in browser, multi-Agent panel, Design Mode, Canvases, Security Review for front-end and full-stack iteration. Need SDK for programmable agents.

Favoring Copilot

Already invested in JetBrains / Visual Studio; want low switching cost for completion and Chat / Agent capabilities. Rely on Microsoft ecosystem and GitHub integration.

Favoring Claude Code

Accustomed to terminal tasks; want Agent to autonomously make multi-step changes to the repo. Value MCP native integration and flexible token-based pricing.
